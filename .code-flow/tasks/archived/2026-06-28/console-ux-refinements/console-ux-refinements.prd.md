## 2. 背景与目标

### 2.1 问题陈述

| 维度 | 内容 |
|------|------|
| **问题描述** | 赛博朋克主题第一版上线后，存在多个交互问题：① 顶部导航栏占用垂直空间且视觉不够沉浸，管理员期望侧边栏导航；② 容器列表每行 5 个操作按钮横向排布拥挤，缺少"更多操作"下拉聚合；③ 表格无分页，容器数量多时页面过长；④ 模型配置页面全局/批量/覆盖三段揉杂，缺乏分块层级；⑤ 审计日志页面与容器列表风格不一致。 |
| **当前替代方案** | 顶部 tab 导航、5 按钮平铺、无分页、LLM 三段平铺。功能可用但交互效率和视觉层次不佳。 |
| **触发原因** | 第一版赛博朋克主题聚焦视觉风格，交互细节需要在第二版打磨。 |

### 2.2 目标与价值

| 维度 | 内容 |
|------|------|
| **核心目标** | 优化控制台交互：纯侧边栏导航、操作下拉聚合、后端分页、LLM 分块排版、审计对齐容器列表风格。 |
| **预期价值** | 提升管理员日常操作效率；统一列表页交互模式；支持容器数量增长（分页）。 |

**成功指标**

| 指标 | 当前值 | 目标值 | 衡量方式 |
|------|--------|--------|----------|
| 容器操作步数 | 5 个按钮平铺，占用 200px+ 宽度 | "更多操作"下拉聚合，1 个按钮 + hover 展开 | 手动验收 |
| 列表加载 | 全量加载所有容器 | 后端分页，每页 20 条 | 手动验收 |
| LLM 页面信息层级 | 三段揉杂，无视觉分隔 | 三块分区（卡片/分隔），功能边界清晰 | 视觉走查 |
| 导航空间 | topbar 占 50px 高 | 侧边栏占 200px 宽，纵向空间释放 | 手动验收 |

## 4. 功能需求

### 4.1 功能清单

| 功能ID | 功能名称 | 功能描述 | 优先级 | 来源用户故事 |
|--------|---------|---------|--------|-------------|
| FEAT-01 | 纯侧边栏导航 | 去掉 topbar，改为左侧固定侧边栏：品牌名、导航菜单（容器/模型配置/审计日志）、用户信息+退出、告警铃铛。主内容区右移 | P0 | US-01 |
| FEAT-02 | 操作按钮下拉聚合 | 容器行内"启动/停止/重启/回收/唤醒"5 个按钮合并为"更多操作"按钮，hover 展开下拉菜单 | P0 | US-02 |
| FEAT-03 | 容器列表后端分页 | 后端 API `/api/v1/containers` 增加 `offset`/`limit` 查询参数，响应增加 `total` 字段；前端表格底部右侧显示分页控件 | P0 | US-03 |
| FEAT-04 | 状态筛选样式统一 | 状态下拉 `select` 样式与赛博朋克主题一致（霓虹边框、暗底、focus 发光） | P1 | US-04 |
| FEAT-05 | LLM 页面三块分区 | 模型配置页改为三大区块：① 全局 LLM 配置 + 保存 ② 连通性测试（终端输出）③ 批量应用到容器 + 单用户覆盖。区块间用卡片或分隔线隔开 | P0 | US-05 |
| FEAT-06 | 审计日志后端分页 + 列表对齐 | 审计 API `/api/v1/audit` 增加 `offset`/`limit`，响应增加 `total`；前端表格样式和分页与容器列表一致 | P1 | US-06 |

#### FEAT-01: 纯侧边栏导航

- **描述**: 移除 `Topbar` 组件，新建 `Sidebar` 组件。侧边栏固定在左侧（宽度约 200px），包含：品牌名（顶部）、导航菜单（中间）、用户信息 + 告警铃铛 + 退出（底部）。主内容区 `margin-left` 偏移。
- **验收标准**:
  - [ ] 侧边栏固定左侧，不随滚动
  - [ ] 品牌名：monospace 字体 + 霓虹青，带 glow text-shadow
  - [ ] 导航菜单项：3 项（容器/模型配置/审计日志），当前页高亮（紫色左边框 + 背景）
  - [ ] 菜单项 hover：青色 glow 过渡
  - [ ] 底部：用户信息（👤 + 用户名）+ 告警铃铛（🔔 + 徽章）+ 退出按钮，纵向排列
  - [ ] 主内容区 `padding-left: 200px`，原 `max-width` 居中取消
  - [ ] `App.tsx` 移除 `<Topbar>`，改渲染 `<Sidebar>` + `<main>`

#### FEAT-02: 操作按钮下拉聚合

- **描述**: 容器行内"启动/停止/重启/回收/唤醒"5 个按钮合并。保留"日志""升级""删除"3 个直接按钮，新增"更多操作"按钮，hover 展开下拉菜单显示 5 个操作。
- **验收标准**:
  - [ ] 容器行操作区：`[更多操作 ▼] [日志] [升级] [删除]`
  - [ ] hover "更多操作" → 下拉菜单弹出（`position: absolute`，z-index 高于表格行）
  - [ ] 下拉菜单 5 项：启动/停止/重启/回收/唤醒，每项 hover 青色 glow
  - [ ] 鼠标移出按钮或菜单 → 菜单关闭（200ms delay 防抖）
  - [ ] 点击菜单项 → 执行操作 + 关闭菜单 + 刷新列表

#### FEAT-03: 容器列表后端分页

- **描述**: 后端 `GET /api/v1/containers?offset=0&limit=20` 返回 `{ code: 0, data: { items: [...], total: 42 } }`。前端表格底部右侧显示"第 1/3 页 共 42 条" + 上一页/下一页按钮。
- **验收标准**:
  - [ ] 后端 `handleListContainers` 解析 `offset`/`limit` query 参数（默认 0/20，最大 100）
  - [ ] `repo.ListUsers` 改为 `ListUsers(offset, limit int) ([]User, int, error)`，SQL 加 `LIMIT ? OFFSET ?` + `SELECT COUNT(*)`
  - [ ] 响应结构：`{ items: [...], total: N }`（保持 `code:0` 外层）
  - [ ] 前端 `api.listContainers(offset, limit)` 传参
  - [ ] 分页控件：`< 上一页  |  第 1/3 页 共 42 条  |  下一页 >`，首页时"上一页"disabled，末页时"下一页"disabled
  - [ ] 前端分页 state：`page`/`pageSize`(20)/`total`；切换页码重新请求 API

#### FEAT-04: 状态筛选样式统一

- **描述**: 容器列表工具栏右侧状态下拉 `select` 样式与全局赛博朋克主题一致。
- **验收标准**:
  - [ ] `select` 元素使用 `--bg-primary` 背景、`--color-text-primary` 文字
  - [ ] `border: 1px solid rgba(0,229,255,0.2)`，focus 时 `box-shadow: var(--border-glow)`
  - [ ] `option` 子项样式与主题一致（浏览器限制，至少保证 select 本身统一）
  - [ ] 与搜索 input 视觉对齐

#### FEAT-05: LLM 页面三块分区

- **描述**: LLM 页面重构为三块分区，用卡片包裹：
  - ① **全局 LLM 配置**（form-grid + 保存按钮）
  - ② **连通性测试**（测试按钮 + 终端输出区）
  - ③ **批量应用 + 单用户覆盖**（容器勾选列表 + 应用按钮 + 单用户覆盖表单）
- **验收标准**:
  - [ ] 三块分区用卡片包裹（`background: var(--bg-panel); border: 1px solid rgba(...)`）
  - [ ] 每块有标题（如"全局配置"/"连通性测试"/"批量应用与覆盖"）
  - [ ] 卡片间距 16px
  - [ ] 卡片 hover 无额外效果（纯容器）
  - [ ] 终端输出区只在有输出/loading/error 时显示

#### FEAT-06: 审计日志后端分页 + 列表对齐

- **描述**: 审计 API 增加分页；前端表格工具栏+分页与容器列表模式一致。
- **验收标准**:
  - [ ] 后端 `GET /api/v1/audit?actor=&offset=0&limit=20` 返回 `{ items: [...], total: N }`
  - [ ] `repo.QueryAudit` 增加 offset/limit 参数，返回 total count
  - [ ] 前端 `api.audit(actor, offset, limit)` 传参
  - [ ] 审计页工具栏：左侧 actor 筛选输入框 + 查询按钮，右侧（空，与容器页对齐布局）
  - [ ] 表格底部右侧分页控件（与 FEAT-03 复用同一组件）
  - [ ] 时间戳 monospace、行 hover 发光等已有样式保留

### 4.2 边缘情况

| 场景 | 预期行为 |
|------|----------|
| 侧边栏菜单项过长 | 文字截断 + title tooltip |
| 容器列表 0 条 | 分页控件隐藏，显示空态"暂无容器" |
| 分页时删除最后一条导致当前页为空 | 自动回退到上一页 |
| "更多操作"下拉超出屏幕底部 | 自动向上弹出（`bottom: 100%`） |
| `/api/v1/containers?offset=99999` | 返回空 items，total 不变 |
| 审计日志无记录 | 分页控件隐藏，显示"暂无审计记录" |
| `limit` 超过 100 | 后端自动截断为 100 |

## 5. 非功能需求

| 指标ID | 类型 | 要求 |
|--------|------|------|
| NFR-PERF-01 | 性能 | 分页查询响应 < 200ms（SQLite `LIMIT OFFSET`） |
| NFR-COMPAT-01 | 兼容性 | 桌面端 ≥1280px；侧边栏固定宽度 200px，剩余宽度给内容区 |
| NFR-SEC-01 | 安全 | 分页 `limit` 参数后端校验上限 100，防超大查询 |
| NFR-REL-01 | 可靠性 | 分页 total count 与 items 查询在同一事务或可接受最终一致性 |

## 6. 范围与边界

| 类别 | 内容 |
|------|------|
| **范围（In Scope）** | ① 侧边栏导航（替换 topbar）② 操作按钮 hover 下拉聚合 ③ 容器/审计后端分页 API + 前端分页控件 ④ LLM 页面三块分区 ⑤ 审计页面工具栏+表格+分页对齐 ⑥ 状态下拉样式统一 |
| **非范围（Out of Scope）** | ① 不引入路由库（react-router）② 不做侧边栏折叠/展开 ③ 不做暗/亮主题切换 ④ 不做移动端适配 ⑤ 不增加容器列表排序功能 ⑥ 不新增 Go 模块依赖 |
| **前置假设** | ① 容器列表已有 `repo.ListUsers()` 可改造 ② 审计已有 `repo.QueryAudit()` 可改造 ③ 后端 `net/http` `r.URL.Query().Get("offset")` 可直接解析 ④ 前端可复用现有 `api.ts` request 封装 |

## 7. 依赖与风险

### 7.1 项目依赖

| 依赖方 | 依赖内容 | 最晚交付时间 | 风险等级 |
|--------|----------|--------------|---------|
| Go 后端 `internal/repo/repo.go` | ListUsers + QueryAudit 改造 | 与前端同步 | 低 |
| Go 后端 `internal/api/containers.go` | handleListContainers 加 offset/limit | 与前端同步 | 低 |
| Go 后端 `internal/api/audit_query.go` | handleAuditQuery 加 offset/limit/total | 与前端同步 | 低 |

### 7.2 风险识别

| 风险ID | 描述 | 影响 | 缓解方案 |
|--------|------|------|---------|
| RISK-01 | 侧边栏纯 CSS 实现，hover 下拉菜单定位可能在不同容器数量时偏移 | 交互异常 | 使用 `position: relative` 父容器 + `absolute` 子菜单，测试多种行数 |
| RISK-02 | 后端分页改造涉及 repo 接口变更，可能影响测试中的 mock | 测试失败 | 同步更新 `fakeDriver` 和 repo 测试 |
| RISK-03 | COUNT(*) + LIMIT 在 SQLite 下对大数据量性能未知 | 当前用户量小（< 1000），风险低 | SQLite WAL 模式下 COUNT 性能可接受 |
