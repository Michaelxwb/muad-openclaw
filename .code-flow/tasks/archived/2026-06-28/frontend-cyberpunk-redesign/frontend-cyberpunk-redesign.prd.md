## 2. 背景与目标

### 2.1 问题陈述

| 维度 | 内容 |
|------|------|
| **问题描述** | 当前控制台界面为功能优先的最小化实现：布局紧凑、视觉单调（暗色但无风格特征）、告警占用列表主区域、操作入口散落。管理员日常使用时缺乏视觉层次和操作效率感。 |
| **当前替代方案** | 现有暗色主题（`styles.css` 约 250 行）已定义 CSS 变量，但无设计系统、无组件复用、无交互动效。功能可用但体验平庸。 |
| **触发原因** | 控制台功能已稳定（容器管理、LLM 配置、审计日志），需要在视觉和交互上匹配产品的技术感定位。 |

### 2.2 目标与价值

| 维度 | 内容 |
|------|------|
| **核心目标** | 将 muad 管理控制台重构为青紫霓虹赛博朋克风格，同时优化关键交互（创建容器、告警查看、工具栏布局），提升管理员操作效率和视觉一致性。 |
| **预期价值** | 提升管理员日常运维的操作效率；建立统一的设计语言，便于后续页面扩展。 |

**成功指标**

| 指标 | 当前值 | 目标值 | 衡量方式 |
|------|--------|--------|----------|
| 容器创建步骤数 | 页面内联表单，无模态隔离 | 点击按钮→弹窗→提交，3 步完成 | 手动验收 |
| 告警可见性 | 占据列表上方区域，挤压表格空间 | 不占主区域，铃铛徽章一键查看 | 手动验收 |
| 视觉一致性 | 无设计系统 | 4 页面统一主题，CSS 变量覆盖全部组件 | 视觉走查 |

| **使用场景** | 管理员登录控制台 → 查看容器运行状态 → 创建/启停/删除容器 → 配置全局 LLM → 查看审计日志。典型会话时长 10-30 分钟，桌面端操作。 |

| 编号 | 用户故事 | 优先级 |

| US-02 | 作为管理员，我希望通过工具栏按钮弹出表单来创建容器，以便不离开列表页面就能快速创建且不占用列表空间 | P0 |

#### FEAT-01: 全局赛博朋克视觉主题

- **描述**: 基于 CSS 变量建立赛博朋克设计系统，覆盖所有页面和组件。
- **色板**:
  - 背景: `#0a0a0f` (深黑) / `#111122` (面板)
  - 主霓虹: `#00f0ff` (青) / `#b347ea` (紫) / `#ff2d95` (粉)
  - 功能色: `#00ff88` (成功) / `#ff4466` (危险) / `#ffaa00` (警告)
  - 文字: `#c8d6e5` (主) / `#6b7d99` (次级)
- **验收标准**:
  - [ ] `:root` CSS 变量覆盖全部颜色、边框、阴影、字体
  - [ ] 按钮/输入框/表格/徽章统一样式，hover 有霓虹发光效果（`box-shadow` + `transition`）
  - [ ] 全局背景使用深色 + 微妙网格纹（CSS `background-image` 渐变模拟）
  - [ ] 标题使用 monospace 或科技感字体
  - [ ] 所有交互元素有 `transition: all 0.2s` 过渡

#### FEAT-02: 容器创建模态框

- **描述**: 容器列表页工具栏"创建容器"按钮点击后弹出模态框，表单字段与原内联表单一致（userId、botId、secret），提交后自动关闭并刷新列表。
- **验收标准**:
  - [ ] 点击"创建容器"弹出模态框，背景半透明遮罩
  - [ ] 表单字段: userId、botId、secret（均为必填），布局清晰，错误提示在字段下方
  - [ ] 提交中按钮 disabled + loading 文案，防止重复提交
  - [ ] 成功: 关闭模态框、刷新列表、显示成功提示
  - [ ] 失败: 模态框保持打开，显示错误信息
  - [ ] 点击遮罩或关闭按钮关闭模态框（不提交）
  - [ ] 模态框样式与赛博朋克主题一致（霓虹边框、暗底）

#### FEAT-03: 用户信息+退出整合

- **描述**: 应用初始化时调用 `GET /api/v1/me` 获取当前用户名，在 topbar 右侧与退出按钮组合显示。
- **验收标准**:
  - [ ] App mount 后调用 `/api/v1/me`，获取 `{ actor: "admin" }`
  - [ ] topbar 右侧显示用户图标/名称 + "退出"按钮，视觉上为一组
  - [ ] 点击退出清除 token 并返回登录页
  - [ ] 加载用户信息失败时不阻塞页面，降级显示"未知用户"

#### FEAT-04: 通知铃铛告警

- **描述**: topbar 右侧告警铃铛图标，定期拉取 `/api/v1/alerts`，有告警时显示红色计数徽章。点击铃铛弹出下拉面板展示告警详情。
- **验收标准**:
  - [ ] 铃铛图标在 topbar 右侧（用户信息旁），每 30 秒轮询 `/api/v1/alerts`
  - [ ] 有告警时铃铛上显示红色数字徽章（告警数量）
  - [ ] 点击铃铛弹出下拉面板，列出告警（级别、用户、消息）
  - [ ] 点击面板外部或再次点击铃铛关闭面板
  - [ ] 无告警时面板显示"暂无告警"
  - [ ] P1 级告警红色边框、P2 黄色、P3 灰色

#### FEAT-05: 列表工具栏重构

- **描述**: 容器列表上方工具栏分为左右两区：左侧操作按钮组（创建容器、重载 Skill），右侧筛选输入框（按 userId 搜索、按状态过滤）。
- **验收标准**:
  - [ ] 工具栏在表格上方，`display: flex; justify-content: space-between`
  - [ ] 左侧: "创建容器"按钮 + "重载 Skill"按钮
  - [ ] 右侧: 搜索输入框（按 userId 过滤）+ 状态下拉筛选（全部/运行中/已停止/异常）
  - [ ] 筛选条件变化时客户端过滤列表（无需请求后端）
  - [ ] 工具栏响应式：宽度不足时右侧筛选换行

#### FEAT-06: Login 页面赛博朋克化

- **描述**: 登录卡片应用赛博朋克主题：霓虹发光边框、背景扫描线动画、标题科技感字体。
- **验收标准**:
  - [ ] 登录卡片有青/紫色 neon glow 边框
  - [ ] 背景有 CSS 动画（扫描线或粒子效果）
  - [ ] 标题字体使用 monospace，"muad 控制台" 加霓虹色
  - [ ] 输入框 focus 时边框发光

#### FEAT-07: LLM 配置页面赛博朋克化

- **描述**: LLM 配置表单和测试结果展示应用赛博朋克主题。
- **验收标准**:
  - [ ] 表单输入框和按钮统一霓虹风格
  - [ ] LLM 测试结果输出区使用终端风格（黑底绿字/青字 monospace）
  - [ ] 保存/测试按钮有发光 hover 效果

#### FEAT-08: 审计日志页面赛博朋克化

- **描述**: 审计日志表格和筛选应用赛博朋克主题。
- **验收标准**:
  - [ ] 表格行 hover 时背景发光过渡
  - [ ] 时间戳使用 monospace 字体
  - [ ] 搜索筛选框统一霓虹风格

### 4.2 边缘情况

| 场景 | 预期行为 |
|------|----------|
| `/api/v1/me` 请求失败 | 降级显示"未知用户"，不阻塞页面渲染 |
| 告警轮询失败 | 保持上次告警数据，铃铛不消失，静默重试 |
| 创建容器模态框打开时刷新页面 | 模态框关闭，表单数据不保留（无 localStorage） |
| 搜索过滤无匹配结果 | 表格显示"无匹配容器"，保留过滤条件可修改 |
| 极端长 userId | 表格列宽自适应，超长截断 + title tooltip |
| 浏览器不支持 CSS `backdrop-filter` | 降级为纯色不透明遮罩 |

## 5. 非功能需求

| 指标ID | 类型 | 要求 |
|--------|------|------|
| NFR-PERF-01 | 性能 | 页面首次渲染 < 2s（含 `/api/v1/me` 调用）；告警轮询间隔 30s |
| NFR-COMPAT-01 | 兼容性 | 支持 Chrome/Edge/Firefox 最近 2 个主版本；仅桌面端（≥1280px 宽） |
| NFR-SEC-01 | 安全 | token 存储和传输方式不变（localStorage + Bearer header），不引入新安全边界 |
| NFR-REL-01 | 可靠性 | 用户信息加载失败不影响页面功能；告警轮询失败不阻塞 UI |

## 6. 范围与边界

| 类别 | 内容 |
|------|------|
| **范围（In Scope）** | ① 全局 CSS 设计系统重写（赛博朋克主题）② 容器创建模态框 ③ topbar 用户信息展示 ④ 告警通知铃铛 ⑤ 列表工具栏左右分区 ⑥ Login/LLM/Audit 页面风格统一 |
| **非范围（Out of Scope）** | ① 不新增后端 API（仅 `/api/v1/me` 已存在即够用）② 不做移动端/平板适配 ③ 不引入第三方 UI 组件库 ④ 不改动后端业务逻辑 ⑤ 不做国际化（i18n）⑥ 不做暗/亮主题切换 |
| **前置假设** | ① `/api/v1/me` 接口稳定可用 ② `/api/v1/alerts` 接口稳定可用 ③ 浏览器支持 CSS Grid/Flexbox/`backdrop-filter`/CSS 动画 |

## 7. 依赖与风险

### 7.1 项目依赖

| 依赖方 | 依赖内容 | 最晚交付时间 | 风险等级 |
|--------|----------|--------------|---------|
| 后端 `/api/v1/me` | 返回当前用户信息 | 已就绪 | 低 |
| 后端 `/api/v1/alerts` | 返回告警列表 | 已就绪 | 低 |

### 7.2 风险识别

| 风险ID | 描述 | 影响 | 缓解方案 |
|--------|------|------|---------|
| RISK-01 | 无设计稿，赛博朋克风格效果依赖纯 CSS 实现 | 视觉效果可能不够精致 | 参考成熟赛博朋克 CSS 色板和效果库，分步迭代 |
| RISK-02 | CSS 变量全局改动可能引入样式回归 | 部分页面显示异常 | 逐页验收，`styles.css` 单文件变更易回滚 |
| RISK-03 | 3 个 npm 安全漏洞（1 moderate / 1 high） | 不影响本次 UI 改动 | 后续单独处理 `npm audit fix` |
